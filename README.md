## Collection Overview

This repository is the main directory for a growing collection of standalone GitHub Pages portfolio themes. Each available theme is maintained in its own repository so visitors can browse the design, open a live demo, review screenshots, and access the source files independently.

The current collection contains **59 theme entries**:

- **58 available themes**
- **1 theme under manual review:** `top-secret`

The main collection repository is:

```text
github-pages-themes
```

Each individual theme is stored in its own repository with a short, readable name such as:

```text
oracle
nightshade
aurora-drift
windows-xp
devhaunt
```

## What the Collection Includes

The collection includes a wide range of portfolio styles, including:

- witchy and moonlit interfaces
- retro desktop themes
- Y2K and old-web designs
- terminal and developer-workspace themes
- space, observatory, telemetry, and mission-control layouts
- colorful creative portfolios
- arcade and game-inspired pages
- editorial, dossier, and archive-style themes
- playful, spooky, and atmospheric designs

Each available theme repository includes:

- a live GitHub Pages demo
- a standalone static website
- a root-level `index.html`
- an empty `.nojekyll` file
- GitHub Pages-safe relative asset paths
- screenshots where available
- a detailed README
- issue and feature-request templates
- GitHub Pages deployment instructions

## How the Repositories Are Organized

The collection uses one main directory repository and separate repositories for the individual themes.

```text
github-pages-themes/
├── collection homepage
├── searchable theme directory
├── screenshot previews
├── links to live demos
├── links to individual theme repositories
└── issue and feature-request forms
```

Each theme repository is a complete standalone project.

```text
oracle/
├── index.html
├── .nojekyll
├── assets/
├── images/
├── README.md
└── additional theme files
```

Updating one theme does not alter the other themes in the collection.

## Using an Individual Theme

To use one of the available themes:

1. Open the theme repository from the directory above.
2. Review the screenshots and live demo.
3. Download or copy the theme files.
4. Replace the default portfolio content with your own information.
5. Keep `index.html` and `.nojekyll` at the repository root.
6. Enable GitHub Pages from the `main` branch and `/(root)` folder.
7. Test the published site on desktop and mobile.

Each individual theme README explains:

- what the theme includes
- which pages are available
- where shared assets are stored
- how to update written content
- how to replace screenshots and images
- how to publish with GitHub Pages
- how to troubleshoot common problems

## GitHub Pages Deployment

To publish a theme with GitHub Pages:

1. Open the theme repository.
2. Select **Settings**.
3. Select **Pages**.
4. Under **Build and deployment**, choose **Deploy from a branch**.
5. Select:

```text
Branch: main
Folder: / (root)
```

6. Select **Save**.
7. Open the published GitHub Pages URL.
8. Test the homepage, navigation, images, styling, and links.

## Important Repository Files

### `index.html`

This is the homepage and GitHub Pages entry file. It must remain at the repository root.

### `.nojekyll`

This is an empty file stored beside `index.html`.

The file should remain completely empty. Its filename is the instruction.

Correct:

```text
.nojekyll
```

Incorrect:

```text
nojekyll
.nojekyll.txt
nojekyll.md
```

### `README.md`

This file explains the theme, available pages, customization options, and GitHub Pages setup.

### `.github/ISSUE_TEMPLATE/`

This folder stores the detailed bug-report and feature-request forms.

## Reporting Issues or Requesting Additions

Use the collection request forms:

[Report a collection issue or request an addition](https://github.com/apursley2012/github-pages-themes/issues/new/choose)

Use the main collection forms for:

- broken gallery links
- missing previews
- filter problems
- layout issues
- documentation errors
- new collection features
- new theme suggestions

For a problem involving one specific theme, open that theme's repository and use its own issue-request form.

When reporting an issue, include:

- the affected theme or page
- the live URL
- the browser and device
- the steps needed to reproduce the problem
- the expected result
- the actual result
- a screenshot or screen recording when possible

## Accessibility and Browser Compatibility

Before publishing a personalized theme, test:

- keyboard navigation
- focus states
- mobile-width layouts
- image alternative text
- readable color contrast
- reduced-motion preferences
- link clarity
- empty and error states

The themes should be tested in current versions of:

- Chrome
- Firefox
- Safari
- Edge
